## 二、骨架屏设计

### 需要添加骨架屏的页面

| 页面 | 优先级 | 说明 |
|------|--------|------|
| **球队详情** (team-detail) | P0 | 访问最频繁，加载内容包括球队信息+成员列表 |
| **能力档位设置** (skill-level) | P0 | 数据量大，需要展示统计卡片 |
| **分组设置** (random-group) | P1 | 包含复杂的选择器状态 |
| **分组结果** (group-result) | P1 | 展示数据量最大，包含多个队伍卡片 |

### 骨架屏设计规范

**通用原则**
- 灰色调：`#F2F2F2` 背景，`#E6E6E6` 占位块
- 圆角统一：16rpx
- 动画效果：微弱的左右渐变扫光（1.5s 循环）

**加载状态切换逻辑**
```
onLoad → 显示骨架屏 → 请求数据 → 数据返回 → 淡入真实内容
```

**防抖策略**
- 数据请求 < 500ms：直接显示内容
- 数据请求 > 500ms：显示骨架屏
- 数据请求 > 3s：显示加载进度或错误提示

## 三、动画效果设计

### 3.1 分组动画（重点）⭐

**动画流程**（总时长约 4 秒）

```
1️⃣ 准备阶段（0-500ms）
   已选球员卡片从各档位区域飞出 → 聚合到屏幕中心

2️⃣ 随机飞舞阶段（500-2000ms）
   卡片在中心旋转、飞舞 → 营造"随机"的视觉效果

3️⃣ 分组落地阶段（2000-3500ms）
   卡片按队伍依次飞向目标区域 → 落地时有轻微弹跳效果

4️⃣ 结果展示阶段（3500-4000ms）
   所有队伍卡片完全展示 → 统计数据淡入
```

### 3.2 页面切换动画

| 场景 | 动画效果 | 时长 |
|------|----------|------|
| 列表 → 详情 | 从右向左滑入 | 300ms |
| 详情 → 列表 | 从左向右滑出 | 300ms |
| 弹窗打开 | 从底部弹起 | 250ms |
| 弹窗关闭 | 向下滑落 | 250ms |
| Tab 切换 | 内容淡入淡出 | 200ms |

### 3.3 按钮交互动画

- **点击反馈**：缩放到 0.95 + 透明度 0.8（100ms）
- **加载状态**：三个点依次跳动
- **成功状态**：背景变绿 + 打勾动画

### 3.4 成功/失败反馈

**成功反馈**
- ✓ Toast 轻提示（顶部滑入，停留 1.5s）
- ✓ 成功弹窗（打勾动画 + 彩带效果）

**失败反馈**
- ✗ Toast 错误提示（红色背景 + 抖动效果）
- ✗ 错误页面（表情 + 重新加载按钮）

## 四、技术方案

### 骨架屏实现

**推荐方案**：自定义组件 + CSS 动画

```
/components/skeleton/
├── skeleton.js
├── skeleton.json
├── skeleton.wxml
├── skeleton.wxss
```

### 动画实现

| 动画类型 | 技术方案 |
|----------|----------|
| 简单过渡动画 | CSS transition |
| 复杂关键帧动画 | CSS animation |
| 分组飞舞动画 | CSS transform + setTimeout |
| 成功/失败反馈 | Lottie（可选） |

**性能优化**
- ✅ 仅动画 `transform` 和 `opacity`
- ✅ 启用 GPU 加速（`translateZ(0)`）
- ✅ 控制同时播放的动画数量
- ✅ 动画结束后清理

## 七、代码示例

### 骨架屏组件使用

```xml
<!-- 页面中使用 -->
<skeleton 
  wx:if="{{loading}}"
  show-header="{{true}}"
  show-list="{{true}}"
  list-count="{{5}}"
/>

<view class="content fade-in" wx:if="{{!loading}}">
  <!-- 真实内容 -->
</view>
```

### 动画样式使用

```xml
<!-- 按钮点击反馈 -->
<button class="btn btn-press">确定</button>

<!-- 列表项依次进入 -->
<view class="list-item-enter delay-{{index * 50}}" 
      wx:for="{{items}}" 
      wx:key="id">
  {{item.name}}
</view>

<!-- Toast 提示 -->
<toast 
  visible="{{toastVisible}}"
  message="操作成功"
  type="success"
/>
```
