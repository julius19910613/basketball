## 一、测试策略

### 测试范围与优先级

| 模块 | 内容 | 优先级 |
|------|------|--------|
| 骨架屏 | 通用组件 + 四页面布局 | P0 |
| 扫光动画 | 1.5s 循环，性能表现 | P0 |
| 分组动画 | 卡片飞舞、聚合、落地 | P0 |
| 基础动画 | 页面切换、弹窗、按钮反馈 | P1 |
| 微交互 | 选中态、Toast、成功弹窗 | P1 |
| 兼容性 | iOS/Android 微信、深色模式 | P1 |
| 性能 | 帧率、内存、低端机 | P1 |

### 测试环境

**iOS 设备：** iPhone 15 Pro Max、iPhone 14、iPhone X、iPhone SE
**Android 设备：** 小米 14、华为 P50 Pro、红米 Note 9
**微信版本：** 8.0.44+（最新）、8.0.40（主流）、8.0.35（最低）

## 二、功能测试要点

### 骨架屏测试

**通用组件**
- ✅ loading=true 显示骨架屏
- ✅ loading=false 显示真实内容
- ✅ 颜色：背景 #F2F2F2，占位 #E6E6E6
- ✅ 扫光动画 1.5s 循环

**各页面验证**
- team-detail：球队信息卡 → 快捷入口 → 成员列表
- skill-level：统计卡 → 快捷操作 → 球员列表
- random-group：统计卡 → 队伍数选择 → 球员选择区
- group-result：统计卡 → 多个队伍卡片

### 分组动画测试

- ✅ 触发：点击"开始分组"
- ✅ 三阶段：飞舞（0.8s）→ 聚合（0.8s）→ 落地（0.9s）
- ✅ 支持 2-5 个队伍
- ✅ 可跳过动画
- ✅ 返回页面时资源释放

### 微交互测试

- ✅ 球员选中：放大 + 边框 + 对勾

## 三、性能测试标准

### 关键指标

| 指标 | 高端机 | 中端机 | 低端机 |
|------|--------|--------|--------|
| 骨架屏加载 | < 100ms | < 150ms | < 200ms |
| 分组动画 FPS | ≥ 60 | ≥ 50 | ≥ 30 |
| 页面滚动 FPS | ≥ 60 | ≥ 55 | ≥ 45 |
| CPU 占用 | < 30% | < 40% | < 50% |
| 内存增量 | < 10MB | < 15MB | < 20MB |

- **PerfDog**：真机性能监控（FPS、CPU、内存）
- **微信开发者工具**：性能分析面板
- **vconsole**：真机调试

## 七、风险与应对

| 风险 | 应对措施 |
|------|----------|
| 低端机性能不达标 | 提供动画降级方案 |
| 兼容性问题多 | 扩大测试覆盖 |
| 用户反馈差 | 增加内测，快速迭代 |

**降级方案：**
- 低端机动画卡顿：简化动画（移除飞舞阶段）
- FPS < 30：禁用动画，直接显示结果
- 内存不足：移除扫光动画
